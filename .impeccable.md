# Unveiling — Frontend Design Context

整理自 2026-05-11,基于 PRD.md §"前端设计"(第 546–728 行) + 用户访谈。
后续设计决策的最终仲裁文件。

## Design Context

### Users

**Primary user**: 一个带着具体问题来的思考者——投资决策、战略疑问、想验证的判断,等等。
他们之前用过 ChatGPT / Perplexity,已经拿到过显而易见的答案,现在想要一个**不同的角度**,
而不是再多一份同温层的肯定。

**Context**: 坐在桌前,愿意为单次分析投入 10–30 分钟。聪明、爱质疑、看重"思想上的意外"
胜过"完整性"。手机端会用但不是核心场景。

**Job to be done**: 把我的问题放到一组刻意远的类比下,告诉我深层结构的对应,以及结论
在哪里成立、哪里破裂——这样我能自己判断信多少、付诸什么行动。

### Brand Personality

- **Voice**: 一位有洞察力的资深分析员,会带着对未知边界的谦逊交付结论。
  不是 chatbot,不是搜索引擎,不是 dashboard。
- **Tone**: 节制、文学、对自己的边界很坦然。"我注意到这件事;张力在这儿;
  到这儿它就不成立了。"
- **三个词**: *Intelligent · Honest · Restrained*
- **情绪目标**: 像读一篇出色的随笔——读完会换一种想法,留下一个尖锐的主意。

### Aesthetic Direction

- **锚定隐喻**: 一份研究随笔 / 印刷品——不是 SaaS dashboard
- **画布**: 奶油底 #faf7f0,与 PRD 一致;不切换深色,不引入玻璃态
- **字体**: 衬线(Source Han Serif SC)用在大标题、结论、引文;
  无衬线(系统 sans)用在 UI 与正文;等宽只用于"机器视角"那一道暗门
- **颜色纪律**:
  - 橙 #ff6b35 是稀缺颜料,只染"张力 / 意外发现 / 当前正在做"
  - 状态色克制(绿/琥珀/灰/锈)只在小圆点和文字里出现,不做大色块
- **运动**: 短(150–600ms),低调,尊重 prefers-reduced-motion;
  无 bounce / elastic 缓动

### Anti-references (要避免长成什么样)

- ChatGPT 结果面板那种"灰底 + 一串聊天泡"
- Perplexity 那种侧栏密度爆炸
- Notion AI summary box 的圆角 + 渐变图标
- Vercel / Linear / Bento-grid 的均匀卡片堆叠 + sparkline
- 任何 cyan-on-dark / 紫蓝渐变 / 玻璃态卡片

### Absolute Bans (impeccable 协议级别的)

1. **不准 `border-left/right: 3px solid <颜色>` 当配色条**。当前代码里
   .tree-node / .narration-bar / .bubble / .sub-question-card / .evidence-item 都中招,要全部重写。
2. **不准 gradient text** (`background-clip: text` + 渐变)。
3. **不准纯黑 / 纯白**——所有"黑色"调一点暖,所有"白色"贴 cream。

### Design Principles

1. **每屏只有一个主焦点**。不能同时让用户的眼睛对四个区域。
2. **发现不能丢**。系统辛辛苦苦找出的意外是这个产品的命脉,UI 不能用 toast 把它扔掉。
3. **等待要显得有意图,不显得焦虑**。分析阶段是"一篇正在写出来的随笔",不是"console 输出"。
4. **结论先于证据,张力先于收敛**。PRD §"结果阶段"的认知节奏:
   一句话回答 → 张力 → 边界 → 收敛 → 意外 → 未尽 → 然后呢
5. **不确定要显式标注**。stuck、降级、缺 killer_evidence——都要看见,不藏。
6. **重读的人最重要**。屏幕上的东西要值得第二次回到这里。
